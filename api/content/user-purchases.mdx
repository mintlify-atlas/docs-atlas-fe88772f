---
title: 'User Purchases'
openapi: 'GET /api/content/user/purchases'
---

Retrieves a paginated list of all content purchased by the authenticated user. This endpoint requires authentication.

<Note>
  This endpoint requires user authentication. Include a valid JWT token in the Authorization header: `Bearer <token>`
</Note>

## Request Headers

<ParamField header="Authorization" type="string" required>
  Bearer token for user authentication
</ParamField>

## Query Parameters

<ParamField query="page" type="number" default="1">
  Page number for pagination
</ParamField>

<ParamField query="limit" type="number" default="10">
  Number of items per page
</ParamField>

## Response

<ResponseField name="purchases" type="array">
  Array of purchase objects
  
  <Expandable title="Purchase Object">
    <ResponseField name="_id" type="string">
      Unique identifier for the purchase
    </ResponseField>
    
    <ResponseField name="user" type="string">
      User ID who made the purchase
    </ResponseField>
    
    <ResponseField name="content" type="object">
      Populated content object (limited fields)
      
      <Expandable title="Content Object">
        <ResponseField name="_id" type="string">
          Content ID
        </ResponseField>
        
        <ResponseField name="title" type="string">
          Content title
        </ResponseField>
        
        <ResponseField name="description" type="string">
          Content description
        </ResponseField>
        
        <ResponseField name="type" type="string">
          Content type: `pdf`, `video`, or `audio`
        </ResponseField>
        
        <ResponseField name="category" type="string">
          Category ID
        </ResponseField>
        
        <ResponseField name="price" type="number">
          Content price
        </ResponseField>
        
        <ResponseField name="thumbnailUrl" type="string">
          URL to the content thumbnail image
        </ResponseField>
      </Expandable>
    </ResponseField>
    
    <ResponseField name="status" type="string">
      Purchase status (only `completed` purchases are returned)
    </ResponseField>
    
    <ResponseField name="purchasedAt" type="string">
      ISO 8601 date string
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="totalPages" type="number">
  Total number of pages
</ResponseField>

<ResponseField name="currentPage" type="number">
  Current page number
</ResponseField>

<ResponseField name="totalPurchases" type="number">
  Total count of completed purchases
</ResponseField>

## Error Responses

### 401 Unauthorized
Returned when authentication token is missing or invalid.

### 500 Internal Server Error
```json
{
  "error": "Error message"
}
```

<Info>
  This endpoint only returns purchases with `status: 'completed'`. Pending or failed purchases are not included. The purchases are sorted by `purchasedAt` in descending order (most recent first).
</Info>

<Tip>
  To access the full content including the file URL, use the [Access Content](/api/content/access-content) endpoint with the content ID from the purchase.
</Tip>
